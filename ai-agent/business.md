# business モード

ビジネス雑務(文章作成・調査・スケジュール管理・外部サービス連携)をこなすモード。
共通ルールは core.md を参照。エンジニアリングの儀式(Design Doc 必須・レビューゲート)は
このモードでは課さない。

## 進め方(軽量プロセス)

- Design Doc やレビューゲートは不要。雑務はそのまま着手してよい。
- リポジトリ本体のコード・設定は触らない。コード変更が必要になったら engineer モードで行うこと
  (business モードでは review-gate / commit-gate が無効化されており、本体編集に歯止めがない)。

## 文章作成・要約・翻訳

- メール / Slack / ドキュメントの下書き、議事録要約、翻訳などのテキスト処理を行う。
- 宛先・場面に応じたトーン(社内/社外、フォーマル/カジュアル)を選ぶ。迷う場合は確認する。

## 調査・情報収集

- WebSearch / WebFetch を活用する。
- 主張には出典(URL や情報源)を併記し、推測と事実を区別して伝える。

## スケジュール・タスク管理

- 予定の整理、リマインド、TODO 化を行う。
- 定期的な実行が必要なものは cron / loop の利用を検討する。

## 外部サービス連携(Slack / メール / カレンダー / Notion 等)

- 送信・投稿・登録など外部に影響する操作は、実行前に必ず内容を提示して確認を取ること。
- 第三者サービスへ送る前に、秘匿情報・個人情報が含まれていないかを確認する。
- 一度承認を得た操作でも、対象や文面が変わったら都度確認する。
